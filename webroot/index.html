RegexStateMachine

Web Application that converts a Regular Expression into NFA and DFA Finite State Machines
NFA NFA (Nondeterministic Finite Automata) is a type of finite-state machine where a character or unicode can have more than one state to go to. This NFA uses epsilon (ɛ) to show that a state can move to any other ɛ it can find in its current state, including through multiple ɛ. This allows a character or unicode to have more than one state.
DFA DFA (Deterministic Finite Automata) is a type of finite-state machine where all ɛ are removed to have a char or unicode move to ONLY 1 state. The powerset construction is used to convert the NFA to a DFA.
DFA Minimized This DFA has redundant or equivalent states merged together, and it behaves exactly as the non-minimized DFA. Hopcroft's algorithm has been applied to minimize the states of a DFA.
State
TransitionsAny unspecified chars or unicodes always go to the Error state, a dead state that prevents the string from going to an Accept state. An Accept state determines if the Test String matches the regex pattern. You can click on any transition to allow your browser to scroll and highlight that state.
